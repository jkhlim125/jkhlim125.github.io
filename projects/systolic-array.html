Systolic array accelerator

Verilog + Python · 4×4 array · VCD + trace verification

Array 4×4Latency 13 vs 12 cycVerify VCD + trace.csv

A complete 4×4 systolic array — controller (FSM IDLE→LOAD→STREAM→DRAIN→COLLECT→DONE), input loader, PE mesh, output collector, testbench — built for observability: every run emits a cycle-level trace.csv that a Python pipeline turns into behavior plots.

13 cyc
Measured latency for the 4×4 array
12 cyc
Theoretical — the extra cycle is control overhead, measured not assumed

Correctness shows up directly in the trace: the per-PE MAC-fire mask advances diagonally (0001 → 0013 → 0137 → 137f → 37fe → 7fec → fec8 …), i.e. the systolic wavefront sweeping across the array. Each PE gates accumulation on operand validity (enable && a_valid && b_valid), so only aligned data contributes.
