LUT vs MAC inference pipeline

Verilog · controlled datapath comparison · event-based latency

Two inference datapaths in RTL — LUT-based and MAC-based — processing the same input under identical control, so the only variable is the datapath. Latency is read from real signal events (out_valid rising edge minus in_valid), not assumed pipeline depth.

Runs showing the identical +1 cycle

The same +1 across all 18 runs, traceable to exactly one pipeline register in the MAC scoring datapath. A small result, but a cleanly controlled and verified one — and a demonstration that latency has to be measured from real events, not inferred from designed depth.
